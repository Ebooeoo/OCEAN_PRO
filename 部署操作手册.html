创建 Supabase 数据库

完全免费 · 约 5 分钟

  1. 打开 Supabase 官网并注册

    访问 https://supabase.com，点击右上角 Start your project，用 GitHub 账号直接登录，无需填写银行卡。

  2. 新建项目

    点击 New Project → 填写项目名称（如 marine-bio）→ 设置数据库密码（记住这个密码！）→ 地区选 Southeast Asia (Singapore) → 点击 Create new project。

    ⚠️ 记住密码！数据库密码等下配置环境变量时要用，创建后无法找回。

  3. 获取数据库连接字符串

    项目创建后（约1分钟），点击左侧 Settings → Database → 找到 Connection string 区域 → 选择 URI 选项卡 → 复制连接字符串。

    # 连接字符串格式如下（YOUR-PASSWORD 替换为你设的密码）：
    postgresql://postgres:[YOUR-PASSWORD]@db.xxxx.supabase.co:5432/postgres
    💡 连接字符串在哪里？Settings → Database → Connection string → URI 选项卡

  4. 初始化数据库表和数据

    点击左侧 SQL Editor → 点击 New query → 把下面的 SQL 全部粘贴进去 → 点击 Run（▶️ 按钮）。

    -- 用户表
    CREATE TABLE IF NOT EXISTS users (
      id SERIAL PRIMARY KEY,
      username VARCHAR(50) NOT NULL UNIQUE,
      password VARCHAR(255) NOT NULL,
      role VARCHAR(20) NOT NULL DEFAULT 'public',
      name VARCHAR(100),
      email VARCHAR(100),
      avatar TEXT,
      status SMALLINT DEFAULT 1,
      created_at DATE DEFAULT CURRENT_DATE
    );

    -- 生态系统表
    CREATE TABLE IF NOT EXISTS ecosystems (
      id SERIAL PRIMARY KEY,
      name VARCHAR(100) NOT NULL,
      type VARCHAR(50),
      description TEXT,
      area VARCHAR(200),
      created_at DATE DEFAULT CURRENT_DATE
    );

    -- 物种表
    CREATE TABLE IF NOT EXISTS species (
      id SERIAL PRIMARY KEY,
      chinese_name VARCHAR(100) NOT NULL,
      latin_name VARCHAR(200),
      phylum VARCHAR(100), class VARCHAR(100),
      "order" VARCHAR(100), family VARCHAR(100),
      genus VARCHAR(100), species VARCHAR(100),
      morphology TEXT, habits TEXT, distribution TEXT,
      longitude DECIMAL(10,6), latitude DECIMAL(10,6),
      protection_level VARCHAR(50) DEFAULT '无',
      endangered_status VARCHAR(10) DEFAULT 'LC',
      image_url TEXT, video_url TEXT, references_text TEXT,
      created_by VARCHAR(100),
      created_at DATE DEFAULT CURRENT_DATE,
      status SMALLINT DEFAULT 1
    );

    -- 观测记录表
    CREATE TABLE IF NOT EXISTS observations (
      id SERIAL PRIMARY KEY,
      title VARCHAR(200) NOT NULL,
      observed_at TIMESTAMP,
      longitude DECIMAL(10,6), latitude DECIMAL(10,6),
      ecosystem_id INT REFERENCES ecosystems(id) ON DELETE SET NULL,
      ecosystem_name VARCHAR(100), observers VARCHAR(200),
      water_temp DECIMAL(5,2), salinity DECIMAL(5,2), depth DECIMAL(8,2),
      weather_condition VARCHAR(50), notes TEXT,
      created_by VARCHAR(100),
      created_at DATE DEFAULT CURRENT_DATE
    );

    -- 关联表
    CREATE TABLE IF NOT EXISTS observation_species (
      id SERIAL PRIMARY KEY,
      observation_id INT NOT NULL REFERENCES observations(id) ON DELETE CASCADE,
      species_id INT NOT NULL REFERENCES species(id) ON DELETE CASCADE,
      species_name VARCHAR(100), count INT DEFAULT 0, behavior VARCHAR(200)
    );

    -- 注册申请表
    CREATE TABLE IF NOT EXISTS user_applications (
      id SERIAL PRIMARY KEY,
      name VARCHAR(100), username VARCHAR(50) NOT NULL UNIQUE,
      email VARCHAR(100), password VARCHAR(255) NOT NULL,
      role VARCHAR(20) DEFAULT 'student',
      status VARCHAR(20) DEFAULT 'pending',
      apply_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
      apply_ip VARCHAR(50), review_remark VARCHAR(200)
    );

    -- 活动日志表
    CREATE TABLE IF NOT EXISTS activity_logs (
      id SERIAL PRIMARY KEY,
      user_id INT, user_name VARCHAR(100),
      action VARCHAR(100), target VARCHAR(200),
      time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, ip VARCHAR(50)
    );

部署后端到 Koyeb

Node.js · 24/7 不休眠 · 约 8 分钟

  1. 注册 Koyeb 账号

    访问 https://app.koyeb.com/auth/signup，点击 Continue with GitHub，用 GitHub 账号直接注册，不需要填写任何支付信息。

  2. 创建新服务

    登录后点击右上角 Create Service → 选择 Web Service。

  3. 连接 GitHub 仓库

    选择 GitHub → 点击 Install GitHub App 授权仓库访问 → 选择 Ebooeoo/OCEAN_PRO 仓库 → 分支选 master。

  4. 配置构建设置

    在构建配置页面填写：

    设置项填写值
    Root directorymarine-backend
    BuilderDockerfile（自动检测）
    Run commandnode src/app.js
    Port3001

  5. 配置环境变量（最关键步骤⚠️）

    在 Environment variables 区域，逐一添加以下变量：

    变量名填写值
    DATABASE_URLpostgresql://postgres:[密码]@db.xxx.supabase.co:5432/postgres
    （从 Supabase 第4步复制的连接字符串）
    JWT_SECRETmarine_gdou_secret_2024_xxxxxxxx（随机字符串，自己定）
    NODE_ENVproduction
    PORT3001
    ⚠️ DATABASE_URL 中的密码：把 [YOUR-PASSWORD] 替换为你在 Supabase 创建项目时设的数据库密码，不要保留方括号。

  6. 选择免费套餐并部署

    在页面底部 Instance 区域选择 Free（免费实例）→ 点击 Deploy。等待约 3-5 分钟构建完成。

  7. 记录后端地址

    部署成功后，Koyeb 会给你一个域名，格式类似：

    https://marine-backend-xxxx.koyeb.app

    访问 https://你的域名/api/health 看到 {"status":"ok"} 就说明后端部署成功了！

    ✅ 后端部署成功！记下这个地址，下一步配置前端要用到。

部署前端到 Netlify

Vue3 静态网站 · 约 5 分钟

  1. 修改前端 API 地址

    在本地，编辑 marine-biodiversity/src/api/http.js，找到 baseURL 那行，把地址改为你的 Koyeb 后端地址：

    // 将这行：
    baseURL: import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api'

    // 改为（或者在 marine-biodiversity/.env.production 文件中设置）：
    VITE_API_BASE_URL=https://你的koyeb域名.koyeb.app/api
    💡 推荐做法：在 marine-biodiversity/ 目录下新建 .env.production 文件，写入上面的环境变量，然后重新构建。

  2. 重新构建前端

    在终端执行：

    cd marine-biodiversity
    npm run build

    构建完成后 dist/ 文件夹就是要上传的内容。

  3. 注册 Netlify 并上传

    访问 https://netlify.com → 用 GitHub 登录 → 在主页找到 "Want to deploy a new site without connecting to Git?" 区域 → 把 dist 文件夹整体拖拽到这里。

    ⚠️ 拖拽整个 dist 文件夹，不是里面的文件，也不是 zip 压缩包。路径：marine-biodiversity/dist/

  4. 配置 SPA 路由重定向

    上传后，在 Netlify 项目的 Site configuration → Build & deploy → Post processing 中，添加重定向规则，或者确认 dist/_redirects 文件存在（内容如下）：

    /* /index.html 200
    📝 若刷新页面出现 404，就是路由重定向没配置，按上面步骤检查。

  5. 获取前端访问地址

    Netlify 会自动分配一个域名，如 https://amazing-ocean-xxxx.netlify.app，就是你的正式访问地址。

常见问题排查

  1. 后端日志报 "数据库连接失败"

    检查 Koyeb 环境变量中 DATABASE_URL 是否正确，密码里有特殊字符需要 URL 编码（如 @ 改为 %40）。

  2. 前端请求 API 报 CORS 错误

    后端 app.js 已配置 cors({ origin: '*' })，若仍报错请检查前端的 VITE_API_BASE_URL 是否以 https:// 开头且没有多余的斜杠。

  3. Netlify 刷新页面 404

    确保 dist 目录下有 _redirects 文件，内容为 /* /index.html 200。

  4. Koyeb 构建失败

    检查 Root directory 是否设置为 marine-backend。Koyeb 会自动检测 Dockerfile，确保仓库里 marine-backend/Dockerfile 文件存在。
